# ADR-004: Maintenance is placed into fixed slots 

## Context
A maintenance window can exceed two hours. 
TimeSlot rejects durations of more than two hours, 
so time slots exceeding two hours cannot exist in our model.

## Decision
The admin can only select predefined time slots. 
If a maintenance task takes more than two hours, one or more time slots must be selected.

## Consequences
A cross-slot case cannot exist (no coding or testing is needed). 
The administrator cannot close the pool for 30 minutes and the maximum slot is two hours.
