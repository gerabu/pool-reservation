# ADR-001: Maintenance scheduling is rejected on conflict

## Context
A maintenance window cannot overlap reservations (core rule 3),
but the spec didn't define who wins when the admin schedules
maintenance over existing reservations.

## Decision
The maintenance request is rejected. The admin sees the conflict
and resolves it manually (cancelling reservations one by one).

## Consequences
No cascade logic between Maintenance and Reservations contexts.
Trade-off: more manual work for the admin in v1.
