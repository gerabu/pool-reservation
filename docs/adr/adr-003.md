# ADR-003: Fixed 2-hour slots instead of free start times

## Context
Residents could either pick from predefined 2-hour slots or choose
arbitrary start times. Free start times turn availability into
interval arithmetic (merging gaps, complements) and require a
range-picker UI — roughly 2 days of work with no architectural
learning value. Real pool usage works in blocks anyway.

## Decision
The day is divided into 6 fixed slots: 8-10, 10-12, 12-14,
14-16, 16-18, 18-20. Reservations and maintenance windows
align to these slots.

## Consequences
Availability is a simple map of 6 slots to a status, and the UI
is a grid. Trade-off: no partial-duration or custom-time
reservations. Free start times deferred to v2 if ever requested.
