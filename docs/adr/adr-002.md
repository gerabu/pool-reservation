# ADR-002: TimeSlots use half-open intervals [start, end)

## Context
Two consecutive slots share a boundary instant (one ends at 16:00,
the next starts at 16:00). The domain needs back-to-back
reservations to work, so touching boundaries must not count
as an overlap.

## Decision
TimeSlots follow the half-open interval convention [start, end):
the end instant does not belong to the slot. Overlap requires
strict inequality: `this.start < other.end && other.start < this.end`.

## Consequences
Back-to-back reservations are allowed. Anyone reading the overlap
logic must know the convention — the tests
"does not overlap a slot that starts exactly when it ends"
document it explicitly.
